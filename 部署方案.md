# new-api Zeabur 完整部署方案

## 1. 部署目标

本方案目标是让线上环境和当前本地 `docker compose up --build` 运行环境保持一致。

| 项目 | 本地当前环境 | Zeabur 线上环境 |
|---|---|---|
| 应用服务 | `new-api` | `new-api` |
| 应用构建 | `Dockerfile` | `Dockerfile` |
| 数据库 | PostgreSQL | PostgreSQL |
| 缓存 | Redis | Redis |
| 应用端口 | `3000` | `3000` |
| 部署方式 | Docker Compose | Zeabur 多服务 |

Zeabur 当前不直接使用本地 `docker-compose.yml` 一键部署，而是把 Compose 里的三个服务拆开创建：

| Compose 服务 | Zeabur 服务 |
|---|---|
| `new-api` | Git Repository + Dockerfile |
| `postgres` | PostgreSQL Database |
| `redis` | Redis Database |

## 2. 本地环境对照

本地 [docker-compose.yml](docker-compose.yml) 里的关键配置如下。

| 配置项 | 本地值 |
|---|---|
| `SQL_DSN` | `postgresql://root:123456@postgres:5432/new-api` |
| `REDIS_CONN_STRING` | `redis://:123456@redis:6379` |
| `TZ` | `Asia/Shanghai` |
| `ERROR_LOG_ENABLED` | `true` |
| `BATCH_UPDATE_ENABLED` | `true` |
| `NODE_NAME` | `new-api-node-1` |

Zeabur 部署时保持同样结构，只把账号、密码、Host 换成 Zeabur 生成的值。

## 3. 部署前准备

| 准备项 | 操作 |
|---|---|
| 代码仓库 | 将当前代码推送到 GitHub 或 GitLab |
| Zeabur 账号 | 登录 `https://zeabur.com` |
| 部署分支 | 确认要部署的分支，例如 `main` |
| 项目目录 | 如果仓库根目录是 `CLIProxyAPI`，Zeabur 的 Root Directory 填 `new-api` |
| 域名 | 可以先用 Zeabur 默认域名，正式上线再绑定自定义域名 |

## 4. 第一步：创建 Zeabur Project

| 步骤 | 操作 |
|---|---|
| 1 | 打开 Zeabur Dashboard |
| 2 | 点击 `New Project` |
| 3 | Project 名称填写 `new-api-prod` |
| 4 | Region 选择离你的主要用户最近的区域 |
| 5 | 创建完成后进入 Project 页面 |

创建完成后，这个 Project 里最终应该有三个服务：

| 服务名 | 类型 |
|---|---|
| `new-api` | Web Service |
| `postgres` | PostgreSQL |
| `redis` | Redis |

## 5. 第二步：创建 PostgreSQL

| 步骤 | 操作 |
|---|---|
| 1 | 在 Project 页面点击 `Add Service` |
| 2 | 选择 `Database` |
| 3 | 选择 `PostgreSQL` |
| 4 | 服务名建议填写 `postgres` |
| 5 | 等待服务创建完成 |
| 6 | 进入 PostgreSQL 服务详情页 |
| 7 | 找到连接信息，复制内网 Host、端口、用户名、密码、数据库名 |

需要整理成下面这个 DSN：

```env
SQL_DSN=postgresql://USER:PASSWORD@HOST:5432/DATABASE
```

示例：

```env
SQL_DSN=postgresql://root:X20go8JS95rdV6w3xG1TDn7FkCpK4ABh@43.155.217.154:30466/zeabur
```

| 字段 | 从哪里拿 |
|---|---|
| `USER` | Zeabur PostgreSQL 用户名 |
| `PASSWORD` | Zeabur PostgreSQL 密码 |
| `HOST` | Zeabur PostgreSQL 内网 Host |
| `5432` | PostgreSQL 端口 |
| `DATABASE` | 数据库名 |

注意：优先使用 Zeabur 的内网 Host，不要用公网连接地址。内网更快，也更稳。

## 6. 第三步：创建 Redis

| 步骤 | 操作 |
|---|---|
| 1 | 回到 Project 页面 |
| 2 | 点击 `Add Service` |
| 3 | 选择 `Database` |
| 4 | 选择 `Redis` |
| 5 | 服务名建议填写 `redis` |
| 6 | 等待服务创建完成 |
| 7 | 进入 Redis 服务详情页 |
| 8 | 找到连接信息，复制内网 Host、端口、密码 |

需要整理成下面这个连接串：

```env
REDIS_CONN_STRING=redis://:PASSWORD@HOST:6379/0
```

示例：

```env
REDIS_CONN_STRING=redis://:872GgBftOK5e04zXUxs93cFCkmlWM1u6@43.155.217.154:31748
```

| 字段 | 从哪里拿 |
|---|---|
| `PASSWORD` | Zeabur Redis 密码 |
| `HOST` | Zeabur Redis 内网 Host |
| `6379` | Redis 端口 |
| `/0` | Redis DB 编号，默认用 `0` |

## 7. 第四步：创建 new-api 服务

| 步骤 | 操作 |
|---|---|
| 1 | 回到 Project 页面 |
| 2 | 点击 `Add Service` |
| 3 | 选择 `Git Repository` |
| 4 | 选择保存当前项目的 GitHub/GitLab 仓库 |
| 5 | 选择要部署的分支，例如 `main` |
| 6 | 如果仓库根目录是 `CLIProxyAPI`，Root Directory 填 `new-api` |
| 7 | Build 方式选择 Dockerfile 或保持 Zeabur 自动识别 |
| 8 | 确认 Dockerfile 路径是 `Dockerfile` |
| 9 | 创建服务 |

项目内置 [Dockerfile](Dockerfile) 会自动完成：

| 阶段 | 作用 |
|---|---|
| `builder` | 构建 default 前端 |
| `builder-classic` | 构建 classic 前端 |
| `builder2` | 编译 Go 后端 |
| 最终镜像 | 启动 `/new-api` 并暴露 `3000` |

## 8. 第五步：配置 new-api 环境变量

进入 `new-api` 服务详情页，找到 `Variables` 或 `Environment Variables`，添加下面变量。

| 变量名 | 示例值 | 是否必须 | 说明 |
|---|---|---|---|
| `SQL_DSN` | `postgresql://USER:PASSWORD@HOST:5432/DATABASE` | 必须 | PostgreSQL 连接串 |
| `REDIS_CONN_STRING` | `redis://:PASSWORD@HOST:6379/0` | 必须 | Redis 连接串 |
| `PORT` | `3000` | 必须 | 服务监听端口，必须写成数字 |
| `TZ` | `Asia/Shanghai` | 必须 | 时区 |
| `ERROR_LOG_ENABLED` | `true` | 必须 | 开启错误日志 |
| `BATCH_UPDATE_ENABLED` | `true` | 必须 | 与本地一致，开启批量更新 |
| `SYNC_FREQUENCY` | `60` | 必须 | 批量同步频率，单位秒 |
| `NODE_NAME` | `new-api-zeabur-1` | 必须 | 节点名称 |
| `SESSION_SECRET` | `换成一串长随机字符串` | 必须 | 会话密钥 |
| `STREAMING_TIMEOUT` | `300` | 建议 | 流式请求超时时间 |
| `TRUSTED_REDIRECT_DOMAINS` | `your-domain.com` | 支付场景必填 | 支付跳转可信域名 |

可以直接复制这个模板，然后替换数据库和 Redis 信息：

```env
SQL_DSN=postgresql://USER:PASSWORD@HOST:5432/DATABASE
REDIS_CONN_STRING=redis://:PASSWORD@HOST:6379/0
PORT=3000
TZ=Asia/Shanghai
ERROR_LOG_ENABLED=true
BATCH_UPDATE_ENABLED=true
SYNC_FREQUENCY=60
NODE_NAME=new-api-zeabur-1
SESSION_SECRET=f3224b30931587a8443dcdab131d99732d1b4fd7cfd8b8d405d16f6bcefe51bc
STREAMING_TIMEOUT=300
TRUSTED_REDIRECT_DOMAINS=your-domain.com
```

`PORT` 必须配置成明确的数字，例如 `3000`。不要写成 `${WEB_PORT}`，否则应用启动时会把 `${WEB_PORT}` 当成端口值，导致 `lookup tcp/${WEB_PORT}: unknown port`。

`SESSION_SECRET` 生成建议：

| 方式 | 示例 |
|---|---|
| macOS 终端 | `openssl rand -hex 32` |
| 在线生成 | 使用随机密码生成器，长度至少 32 位 |

## 9. 第六步：设置端口和域名

| 配置项 | 操作 |
|---|---|
| 服务端口 | 确认 Zeabur 暴露端口为 `3000` |
| 默认域名 | Zeabur 会自动分配一个 HTTPS 域名 |
| 自定义域名 | 在 `Domains` 中绑定自己的域名 |
| DNS | 按 Zeabur 提示添加 CNAME 或 A 记录 |
| HTTPS | Zeabur 会自动签发证书 |

部署完成后访问：

```text
https://你的域名
```

健康检查地址：

```text
https://你的域名/api/status
```

预期结果：

| 地址 | 预期 |
|---|---|
| `/` | 能打开 new-api 页面 |
| `/api/status` | 返回成功状态 |

## 10. 第七步：触发部署

| 步骤 | 操作 |
|---|---|
| 1 | 确认 PostgreSQL 和 Redis 都已运行 |
| 2 | 确认 `new-api` 环境变量已保存 |

日志里重点看这些信息：

|---|---|

| `using PostgreSQL as database` | 数据库类型正确 |

| Redis 无报错 | Redis 连接正常 |
| 服务监听 `3000` | 应用启动正常 |

## 12. 第九步：验证 API 调用

| 步骤 | 操作 |
|---|---|
| 1 | 进入管理后台 |
| 2 | 添加一个可用渠道 |
| 3 | 配置模型和倍率 |
| 4 | 创建用户 API Key |
| 5 | 用 API Key 请求接口 |

测试命令模板：

```bash
curl https://你的域名/v1/chat/completions \
  -H "Authorization: Bearer 你的_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "你的模型名",
    "messages": [
      { "role": "user", "content": "hello" }
    ]
  }'
```

| 结果 | 说明 |
|---|---|
| 返回模型回复 | 部署成功 |
| 返回无可用渠道 | 后台渠道或模型配置不完整 |
| 返回额度不足 | 用户额度或订阅配置不完整 |
| 返回认证失败 | API Key 错误 |

## 13. 支付回调配置

如果启用支付宝当面付、Stripe、Creem 等支付能力，必须使用公网 HTTPS 域名。

| 支付能力 | 回调要求 |
|---|---|
| 支付宝当面付 | 支付宝后台和 new-api 后台配置同一个 HTTPS 回调地址 |
| Stripe | Stripe Dashboard 配置 webhook 地址和 webhook secret |
| Creem | Creem 后台配置 webhook 地址和 webhook secret |
| 易支付 | 配置支付网关地址、商户 ID、密钥和回调地址 |

支付宝当面付路径：

| 类型 | 路径 |
|---|---|
| 余额充值通知 | `/api/user/alipay-f2f/notify` |
| 余额充值返回 | `/api/user/alipay-f2f/return` |
| 订阅购买通知 | `/api/subscription/alipay-f2f/notify` |
| 订阅购买返回 | `/api/subscription/alipay-f2f/return` |

完整 URL 示例：

```text
https://your-domain.com/api/user/alipay-f2f/notify
https://your-domain.com/api/user/alipay-f2f/return
https://your-domain.com/api/subscription/alipay-f2f/notify
https://your-domain.com/api/subscription/alipay-f2f/return
```

## 15. 常见问题处理

| 问题 | 原因 | 处理方式 |
|---|---|---|
| 页面 502 | 服务没启动或端口不对 | 确认端口是 `3000`，查看 Zeabur 日志 |
| `lookup tcp/${WEB_PORT}: unknown port` | `PORT` 被配置成了 `${WEB_PORT}` 字符串 | 将 `PORT` 改成 `3000`，然后重新部署 |
| 数据库连接失败 | `SQL_DSN` 写错 | 确认格式是 `postgresql://USER:PASSWORD@HOST:5432/DATABASE` |
| Redis 连接失败 | `REDIS_CONN_STRING` 写错 | 确认格式是 `redis://:PASSWORD@HOST:6379/0` |
| 构建失败 | Dockerfile 构建阶段失败 | 查看 build log，定位前端 build 或 Go build |
| 初始化失败 | 数据库迁移失败 | 查看 PostgreSQL 日志和 new-api 启动日志 |
| 登录失败 | 默认账号或初始化账号错误 | 使用初始化账号登录，兜底 root 登录后立刻改密码 |
| API 返回无渠道 | 后台未配置渠道 | 添加渠道并启用模型 |
| 支付回调失败 | 域名或 webhook 配置错误 | 确认 HTTPS、回调路径、支付平台白名单 |

## 16. 回滚方案

| 场景 | 处理方式 |
|---|---|
| 新版本构建失败 | Zeabur 通常保持上一版运行，修复后重新部署 |
| 新版本启动失败 | 在 Zeabur 回滚到上一条成功部署记录 |
| 数据库迁移异常 | 停止继续部署，先从 PostgreSQL 备份恢复 |
| Redis 配置异常 | 修正 `REDIS_CONN_STRING` 后重新部署 |
| 支付配置异常 | 关闭对应支付开关，修正后再启用 |
